# YouTube_Traking 개발기획안

## 1. 문서 목적

이 문서는 `YouTube_Traking` 프로그램 개발 전에 기능 범위, 사용자 흐름, 데이터 구조, 운영 방식, 배포 기준을 정리하기 위한 기획안입니다.

현재 단계에서는 바로 코딩하지 않고, 개발 방향과 요구사항을 먼저 확정하는 것을 목표로 합니다.

## 2. 프로젝트 개요

`YouTube_Traking`은 HD투어존의 기존 고객 또는 미체결 DB 고객에게 YouTube 홍보영상을 발송한 뒤, 고객별 영상 반응 로그를 관리하기 위한 Windows 전용 데스크톱 프로그램입니다.

프로그램은 사내 직원이 사용하는 업무 도구로 제작하며, Python과 PySide6 기반으로 개발한 뒤 Windows 실행파일 형태로 배포합니다.

이 프로그램의 최종 목적은 영상에 관심을 보인 고객을 선별하여 아웃바운드 TM 대상자로 활용하는 것입니다.

## 3. 확정된 개발 기준

- 사용자: 한국인 직원
- 문서 언어: 한국어
- 대상 운영체제: Windows 전용
- 개발 언어: Python
- 화면 프레임워크: PySide6
- 배포 방식: Windows 실행파일
- DB 설정 방식: Google Drive 기반 `Config_DB.db`
- DB 연결 공통 모듈: `DB_conn.py`
- 사원 로그인 공통 모듈: `employee_login.py`
- 사원 인증 방식: 사번/비밀번호 로그인

## 4. 공통 인프라 기준

### 4.1 DB 연결

- DB 서버 정보, 계정, 비밀번호, 연결 문자열은 프로그램 코드에 직접 하드코딩하지 않습니다.
- `config.txt`에서 `google_doc_key`를 읽습니다.
- 로컬 `DB/Config_DB.db`가 없으면 Google Drive에서 내려받는 방식을 사용합니다.
- `Config_DB.db`의 `DBCON` 테이블에서 기본적으로 `Name='HD_MSSQL'` 설정을 읽습니다.
- MSSQL 연결은 `pyodbc`를 사용합니다.
- 모든 DB 연결은 `DB_conn.py`를 통해 처리합니다.

### 4.2 사원 로그인

- 로그인 기능은 `employee_login.py`에 공통으로 둡니다.
- PySide6 화면에서는 `LoginDialog(db_config)`를 사용합니다.
- UI 없는 인증이 필요할 때는 `authenticate_employee(db_config, sabun, password)`를 사용합니다.
- 로그인 성공 사용자는 `LoginUser(sabun, saname)` 형태로 다룹니다.
- 비밀번호와 로그인 조회 결과는 로그나 화면에 노출하지 않습니다.

### 4.3 상담이력 저장

상담이력 저장 방식은 고객 구분에 따라 다릅니다.

#### 4.3.1 기존 회원 상담이력 저장

회원 상담이력은 아래 저장 프로시저를 호출해 남깁니다.

```sql
dbo.usp_frmI001Memo_InsertN
```

호출 변수 기준은 아래와 같습니다.

| 변수 | 값 |
| --- | --- |
| `i_Type` | `1` 고정 |
| `MemberNo` | 입력된 회원코드 |
| `Charge_ID` | 처리자 ID, 로그인 사번 |
| `TMemo` | 상담내용 |
| `TelCall_Gu` | 상담구분 |

이 규칙은 TM 진행 결과 저장, 고객별 상담 메모 저장, 발송 후 후속 상담 기록에서 공통으로 사용합니다.

#### 4.3.2 미체결 고객DB 상담이력 저장

아직 계약이 체결되지 않은 고객DB에 대한 상담이력은 프로시저가 아니라 `TM_M_Memo` 테이블에 직접 `INSERT`합니다.

```sql
INSERT INTO TM_M_Memo
    (
    Note_Date,
    Note_time,
    Charge_ID,
    ID,
    TMemo,
    Save_Date,
    Modi_Date,
    TelCall_Gu,
    TelCall_ErrGu,
    TelCall_Mem,
    Memo_GuBun,
    ProDate,
    M_Submit,
    M_PayNum,
    ProTime
    )
VALUES
    (
    LEFT(Convert(VarChar(30),GETDATE(),120),10),
    Substring(Convert(VarChar(30),GETDATE(),120),11,10),
    %s, /* 사번 */
    %s, /* 회원코드 */
    %s, /* 상담내용 */
    Convert(VarChar(30),GETDATE(),120),
    CONVERT(VARCHAR(30),GETDATE(),120),
    %s, /* 상담구분 */
    '',
    '',
    '2',
    '',
    'OTHER',
    0,
    ''
    )
```

입력값 기준은 아래와 같습니다.

| 항목 | 값 |
| --- | --- |
| `Charge_ID` | 처리자 ID, 로그인 사번 |
| `ID` | 입력된 회원코드 |
| `TMemo` | 상담내용 |
| `TelCall_Gu` | 상담구분 |
| `Memo_GuBun` | `2` 고정 |
| `M_Submit` | `OTHER` 고정 |
| `M_PayNum` | `0` 고정 |

상담이력 저장 기능을 만들 때는 대상 고객이 기존 회원인지 미체결 고객DB인지 먼저 구분한 뒤, 각각의 저장 방식을 적용합니다.

### 4.5 알림톡용 영상 썸네일 이미지

카카오 알림톡에 영상 URL을 보낼 때는 고객이 클릭하고 싶도록 영상 썸네일 이미지를 함께 사용합니다.

썸네일 이미지는 아래 기준으로 만듭니다.

- YouTube 영상의 실제 썸네일 이미지를 사용합니다.
- 이미지 중앙에 재생 버튼 `▶`을 오버랩합니다.
- 재생 버튼은 반투명 어두운 원형 배경 위에 흰색 삼각형으로 표시합니다.
- 고객이 “영상 재생 가능 이미지”로 바로 인식할 수 있게 만듭니다.
- 생성된 이미지는 카카오 알림톡 이미지 또는 랜딩 페이지 대표 이미지로 활용합니다.

테스트 영상 기준으로 생성한 예시 파일은 아래 경로에 둡니다.

```text
assets/images/kakao_video_play_thumbnail.png
```

## 6. 업무 배경

HD투어존은 기존 고객 또는 미체결 DB 고객에게 YouTube 홍보영상을 발송하고, 영상을 받은 고객의 관심도를 확인한 뒤 아웃바운드 TM을 진행할 예정입니다.

발송 대상 고객은 관리자가 선정합니다. 관리자는 엑셀에서 고객의 회원코드 목록을 복사한 뒤 프로그램에 붙여넣어 발송 대상을 생성합니다.

관리하고 싶은 주요 반응 로그는 아래와 같습니다.

- 고객이 영상을 열람했는지 여부
- 고객별 영상 재생 횟수
- 고객별 시청 시간 또는 시청 유지 정도
- 고객별 반복 재생 여부
- 좋아요 등 관심 표현 여부
- 영상 반응 이후 TM 진행 여부

단, 일반 YouTube 링크만 고객에게 보내는 방식으로는 YouTube가 고객별 시청 시간, 재생 횟수, 좋아요 여부를 외부 시스템에 직접 제공하지 않습니다. 따라서 고객별 추적을 위해서는 고유 추적 링크, 중간 랜딩 페이지, 자체 로그 수집 방식 등을 함께 검토해야 합니다.

## 9. 발송 대상 생성 방식

1차 버전에서는 관리자가 선정한 회원코드 목록을 엑셀에서 복사해 프로그램에 붙여넣는 방식을 사용합니다.

예상 흐름은 아래와 같습니다.

1. 관리자가 엑셀에서 회원코드 열을 복사합니다.
2. 프로그램의 발송 대상 생성 화면에 붙여넣습니다.
3. 프로그램은 줄바꿈, 탭, 쉼표 등으로 구분된 값을 회원코드 목록으로 정리합니다.
4. 중복 회원코드를 제거합니다.
5. DB에서 회원코드 기준으로 고객 정보를 조회합니다.
6. 조회되지 않은 회원코드와 중복 회원코드를 관리자에게 표시합니다.
7. 관리자가 확인 후 발송 대상 목록을 확정합니다.
8. 확정된 대상 고객별로 고유 추적 링크를 생성합니다.

검토할 예외 상황은 아래와 같습니다.

- 빈 값이 포함된 경우
- 같은 회원코드가 여러 번 붙여넣어진 경우
- 존재하지 않는 회원코드가 포함된 경우
- 발송 제외 대상 고객이 포함된 경우
- 이미 같은 영상 발송 대상에 등록된 고객이 포함된 경우
- 엑셀에서 여러 열을 복사한 경우 회원코드 열을 어떻게 판단할지

- API 키 관리 방식

- YouTube 직접 링크 발송 방식과 중간 랜딩 페이지 방식 비교
- 영상 재생 이벤트를 자체적으로 기록할 수 있는지 여부
- 수집 주기
- 수집 대상
- 수집 실패 시 재시도 방식
- 수집 이력 저장 방식

## 13. 배포 및 운영 기준

- 직원에게는 Windows 실행파일 형태로 배포합니다.
- 사용자 PC에는 별도 Python 개발 환경이 필요하지 않도록 합니다.
- `Config_DB.db`는 로컬 캐시로 사용하되, 원본 설정은 Google Drive 기준으로 관리합니다.
- 실제 운영 전 실행파일 빌드, 로그인, DB 연결, YouTube 데이터 수집을 반드시 검증합니다.
